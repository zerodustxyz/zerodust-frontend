# Error Handling

The SDK uses typed errors that can be caught and inspected.

## Error classes

```typescript
import {
  ZeroDustError,
  BalanceTooLowError,
  QuoteExpiredError,
  NetworkError,
  TimeoutError,
  ChainNotSupportedError,
  InvalidAddressError,
  SignatureError,
  BridgeError,
  isZeroDustError,
} from '@zerodust/sdk';
```

## Handling errors

```typescript
try {
  const quote = await zerodust.getQuote({ ... });
} catch (error) {
  if (isZeroDustError(error)) {
    switch (error.code) {
      case 'BALANCE_TOO_LOW':
        console.log('Balance too low to sweep');
        break;
      case 'CHAIN_NOT_SUPPORTED':
        console.log('Chain not supported');
        break;
      case 'QUOTE_EXPIRED':
        console.log('Quote expired, get a new one');
        break;
      case 'NETWORK_ERROR':
        console.log('Network error, retry');
        break;
      default:
        console.log('Error:', error.message);
    }
  }
}
```

## Error codes

| Code | Description |
|------|-------------|
| `BALANCE_TOO_LOW` | Balance insufficient for sweep |
| `QUOTE_EXPIRED` | Quote has expired (60s limit) |
| `CHAIN_NOT_SUPPORTED` | Chain ID not supported |
| `INVALID_ADDRESS` | Invalid Ethereum address |
| `NETWORK_ERROR` | Network request failed |
| `TIMEOUT` | Request timed out |
| `BRIDGE_UNAVAILABLE` | Cross-chain bridge unavailable |
| `SERVICE_UNAVAILABLE` | ZeroDust service is down |
| `INVALID_SIGNATURE` | Invalid EIP-712 signature |
| `EIP7702_INVALID_SIGNATURE` | Invalid EIP-7702 authorization |
